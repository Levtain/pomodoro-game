# Agent WorkFlow - Agent工作流

## 🎯 概述
Agent工作流用于定义每个Agent的工作循环，从收集到生成、自审核、达到审核标准后生成报告。

## 🧠 核心步骤
### Step 1：收集
- Agent读取：工作规划、工作要求
- Agent收集：相关资源、文档、示例代码
- Agent记录：收集过程和结果

### Step 2：生成
- Agent根据工作规划生成：
  - 代码
  - 文档
  - 配置
  - 数据

- Agent生成时确保：
  - 符合工作要求
  - 遵循审核标准
  - 生成必要的文档

### Step 3：自审核
- Agent自检：
  - [ ] 代码符合规范
  - [ ] 测试全部通过
  - [ ] 文档完整
  - [ ] 满足审核标准

- 如果通过 → 进入Step 4
- 如果不通过 → 返回Step 2，调整后继续

### Step 4：达到审核标准
- Agent检查：
  - [ ] 所有功能完成
  - [ ] 所有测试通过
  - [ ] 所有文档齐全
  - [ ] 满足审核标准

- 如果通过 → 生成任务报告
- 如果不通过 → 继续迭代，回到Step 2或3

### Step 5：生成任务报告
- Agent生成任务报告（使用报告模板）
- 任务报告包含：
  - 完成情况
  - 交付物清单
  - 遇到的问题
  - 下一步建议

- Agent发送任务报告到主频道

## 📊 Agent工作流图
```
收集 → 生成 → 自审核 → 通过？→ 否 → 生成 → ...
        ↓ 是
    达到审核标准 → 通过？→ 否 → 自审核 → ...
        ↓ 是
    生成任务报告
```

## 🎯 核心规则
- **少量上下文**：Agent只读工作规划（1-2页）、工作要求（1页）、审核标准（1页）
- **文档优先**：每一步都生成文档（收集文档、生成文档、审核文档、报告文档）
- **迭代循环**：自审核或达到审核标准不通过时，继续迭代
- **报告模板**：必须使用统一的报告模板

## 🚀 开始使用
每个任务分支都按照这个工作流执行。
