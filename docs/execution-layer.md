# Execution Layer - 执行层（事前确认）

## 🎯 概述
执行层用于规划具体任务和代码编写，主要解决"如何做"的问题。通过事前确认，明确任务、资源、预期结果和验证方法。

## 🧠 核心步骤
### Step 1：明确具体任务
- 任务是什么？[具体描述]
- 属于哪个用户故事？
- 优先级是什么？

### Step 2：明确资源
- 需要的文件：[列表]
- 需要的数据：[列表]
- 需要的账号/API：[列表]

### Step 3：明确预期结果
- 代码：[语言、框架]
- 文档：[README、API文档]
- 测试：[测试用例]

### Step 4：明确验证方法
- 如何验证代码运行？
- 如何验证功能正确？
- 如何验证文档齐全？

### Step 5：最终确认
- 确认任务、资源、预期、验证
- 主人确认后开始执行

## 📋 任务模板
```
任务N：[名称]
- 所属用户故事：[名称]
- 优先级：[高/中/低]
- 预计时间：[X小时]
- 所需资源：[文件、数据、账号等]
- 预期结果：[代码、文档、测试]
- 验证方法：[如何验证]
```

## 🚀 开始使用
如果是具体任务（代码编写、配置文件），使用执行层。
