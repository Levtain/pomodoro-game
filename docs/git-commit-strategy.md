# Git Commit Strategy - Git提交策略

## 🎯 概述
定义Git提交的规范，确保每个任务分支完成后，都要提交产物和工作报告，方便追踪和回顾。

## 🧠 核心原则
- **每次完成一个任务分支，都要提交**
- **必须提交：产物（代码、文档、配置）和工作报告**
- **提交信息必须说明：为什么提交（完成该任务分支）**
- **工作报告必须包含：工作过程、问题、总结、工作时长**

## 📋 提交类型
- **feat** - 新功能
- **fix** - 修复Bug
- **docs** - 文档变更
- **chore** - 构建/工具变更
- **refactor** - 代码重构
- **test** - 测试相关
- **style** - 代码风格

## 📊 提交信息规范
### 提交1：产物提交
```markdown
类型：feat / fix / docs / chore / refactor / test / style

范围：[任务分支名称]

标题：[简要描述]

详细：
- 任务分支：[名称]
- 完成时间：[时间]
- 产物清单：
  - [文件1]
  - [文件2]
  - [文档1]

关联问题：#[issue编号] (如果适用)
```

### 提交2：工作报告提交
```markdown
类型：docs

范围：[任务分支名称]

标题：工作报告

详细：
- 任务分支：[名称]
- 完成时间：[时间]
- 工作时长：[X小时]
- 产出：[描述]
- 遇到的问题：[问题描述]
- 解决方案：[解决方法]
- 下一步：[下一个任务分支]
```

## 🚀 使用流程
### 每个任务分支完成后
1. **提交产物**
   ```bash
   cd /root/clawd
   git add [产物文件]
   git commit -m "feat: [任务分支名称] - [简要描述]"
   ```

2. **提交工作报告**
   ```bash
   cd /root/clawd
   git add [工作报告文件]
   git commit -m "docs: [任务分支名称] - 工作报告"
   ```

3. **推送到GitHub**
   ```bash
   cd /root/clawd
   git push
   ```
