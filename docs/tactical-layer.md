# Tactical Layer - 战术层（敏捷迭代）

## 🎯 概述
战术层用于规划里程碑内的功能开发，主要解决"怎么做"的问题。通过敏捷迭代，将里程碑分解为用户故事，逐个实现并验证。

## 🧠 核心步骤
### Step 1：明确当前里程碑
- 当前里程碑：[名称]
- 目标：[描述]
- 交付物：[列表]

### Step 2：分解为用户故事
- 用户故事1：[名称]
- 用户故事2：[名称]
- 用户故事3：[名称]

### Step 3：明确每个用户故事的要求
- 用户价值：[对用户的价值]
- 实现方式：[如何实现]
- 验收标准：[如何验证]

### Step 4：明确迭代计划
- 优先级：[哪个故事先做]
- 预计时间：[X小时]
- 依赖关系：[是否依赖其他故事]

### Step 5：确认迭代计划
- 向主人确认迭代计划
- 确认优先级和时间预算

## 📋 用户故事模板
```
用户故事N：[名称]
- 用户价值：[对用户的价值]
- 实现方式：[如何实现]
- 验收标准：[如何验证]
- 预计时间：[X小时]
```

## 🚀 开始使用
如果是里程碑内的功能开发，使用战术层。
