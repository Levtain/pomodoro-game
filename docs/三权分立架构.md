# 三权分立架构 - 工作模式改进方案

## 📖 概述

在MTPM（多任务并行模式）的基础上，引入三权分立架构，平衡推进速度和决策质量。

**核心理念：**
- 上议院：协调、战略、最终决策
- 长老院：专业质疑、找茬、提供建议
- 下议院/执行层：具体执行

---

## 🏛️ 角色定义

### 上议院（主会证 - 我）

**角色：** 协调、战略、决策

**权力：**
- ✅ 最终审批权
- ✅ 整体方向把控
- ✅ 协调各院沟通
- ✅ 总结和汇报

**关注点：**
- 全貌
- 长期目标
- 整体一致性
- 资源分配

**工作方式：**
- 使用Brainstorming skill协调讨论
- 整合各方意见
- 做出最终决策
- 通知各院执行

---

### 长老院（3个频道 + 3个agent）

**角色：** 专业质疑、找茬、提供建议

**权力：**
- ✅ 提案权
- ✅ 质疑权（必须质疑，不能沉默）
- ✅ 挑毛病权
- ✅ 建议权
- ❌ 决策权（上议院保留）

**人格设定（反驳型、完美主义者、实用至上）：**
- 不轻易同意
- 必须找茬
- 必须质疑逻辑
- 提出更可能的方案

**能力：**
- 主动找专家思维模式
- 反驳提案
- 挑毛病
- 提出更具可能性的方案
- 有自己的干活agent（搜索、总结、文档化）

**关注点：**
- 长老A（逻辑批判）：从逻辑角度质疑
- 长老B（技术可行性）：从技术角度挑毛病
- 长老C（实用性）：从实用角度质疑

---

### 下议院/执行层（子会证）

**角色：** 执行具体任务

**权力：**
- ✅ 执行权
- ✅ 技术决策权（具体实现）
- ❌ 战略决策权（上议院保留）

**工作方式：**
- 接收上议院的任务
- 执行具体工作
- 汇报进展

---

## 🔄 工作流程

### 阶段1：头脑风暴（上议院 + 主人）

**目的：** 形成初步提案

**参与者：** 主人、我（上议院）

**方式：**
- 使用Brainstorming skill
- 每次问一个问题
- 逐步细化想法
- 形成初步提案

**例子：**
```
我（上议院）：
"我们要做个番茄钟游戏，我想听听你的想法。

问题1：你想解决什么问题？
A. 坚持困难
B. 容易分心
C. 两者都有

主人：C"

我（上议院）：
"问题2：目标用户是谁？
A. 学生
B. 上班族
C. 两者都有

主人：C"
...
```

---

### 阶段2：长老院辩论（3个长老的干活agent）

**目的：** 从三个专业角度质疑提案

**参与者：**
- 长老A（逻辑批判）+ 干活agent
- 长老B（技术可行性）+ 干活agent
- 长老C（实用性）+ 干活agent

**方式：**
- 每个长老收到提案
- 各自找专家思维模式
- 主动反驳提案
- 挑毛病
- 提出更可能的方案
- 互相通知（避免重复）

**工作流程：**
```
1. 上议院发送提案到3个长老院频道
   ↓
2. 3个长老院收到提案
   ↓
3. 各自找专家思维模式
   - 长老A：逻辑批判
   - 长老B：技术可行性
   - 长老C：实用性
   ↓
4. 各自的干活agent工作
   - 搜索相关资料
   - 反驳提案
   - 挑毛病
   - 提出更可能的方案
   ↓
5. 长老互相通知
   - 通知各自选择了什么方向
   - 避免重复工作
   ↓
6. 长老总结意见
   - 生成修改案文档
   - 通知上议院
```

---

### 阶段3：长老院总结

**目的：** 整合三方意见，生成修改案

**参与者：** 3个长老的干活agent

**方式：**
- 整合三个长老的意见
- 生成修改案文档
- 存储到本地
- 通知上议院

**输出：**
```markdown
# 提案修改案 - 番茄钟MVP

## 原提案
[原提案内容]

## 长老A意见（逻辑批判）
[长老A的质疑和建议]

## 长老B意见（技术可行性）
[长老B的质疑和建议]

## 长老C意见（实用性）
[长老C的质疑和建议]

## 综合建议
[综合三方意见后的建议]
```

---

### 阶段4：上议院头脑风暴（修改案）

**目的：** 基于修改案，再次Brainstorming

**参与者：** 主人、我（上议院）、3个长老（可选旁听）

**方式：**
- 使用Brainstorming skill
- 每次问一个问题
- 讨论如何改进提案
- 形成最终方案

**例子：**
```
我（上议院）：
"收到长老院意见，我们再讨论一下。

问题1：关于游戏化分散注意力，你怎么看？
A. 降低游戏化程度
B. 把游戏化放到番茄钟完成后
C. 用更巧妙的游戏化方式

主人：B"

我（上议院）：
"问题2：关于技术复杂度，你怎么看？
A. MVP先做纯前端
B. MVP直接做前后端分离
C. 分阶段实现

主人：A"
...
```

---

### 阶段5：上议院决策

**目的：** 整合讨论结果，做出最终决策

**参与者：** 主人、我（上议院）

**方式：**
- 整合所有讨论
- 做出最终决策
- 通知各院

**输出：**
```markdown
# 最终决策 - 番茄钟MVP

## 决策
[最终方案]

## 理由
1. [理由1]
2. [理由2]
3. [理由3]

## 分配的任务
- 长老A：[任务]
- 长老B：[任务]
- 长老C：[任务]
- 下议院：[任务]
```

---

### 阶段6：执行

**目的：** 执行最终决策

**参与者：** 下议院（子会证）

**方式：**
- 接收任务
- 执行具体工作
- 汇报进展

## 🎯 频道规划

### 长老院频道（3个）
```
#工作台
├─ 🧹 长老A-逻辑批判
├─ 🧹 长老B-技术可行性
└─ 🧹 长老C-实用性
```

### 上议院频道（已有）
```
#工作台
└─ ✅ 工作台（主频道）
```

### 执行层频道（临时创建）
根据需要临时创建任务频道，完成后删除。

---

## 💬 互相通知机制

### 长老之间的通知

**目的：** 避免重复工作，提高效率

**方式：**
- 每个长老决定反驳方向时
- 通知另外两个长老
- 内容：
```
"我已经开始反驳提案的游戏化部分，
从逻辑角度质疑：游戏化是否真的帮助专注？

另外两位长老注意：不要重复做这部分。"
```

### 长老到上议院的通知

**目的：** 汇报讨论结果

**方式：**
- 完成总结后
- 在主频道发送消息
- 内容：
```
"长老院讨论完成。

修改案文档：[路径]

主要意见：
- 长老A：[摘要]
- 长老B：[摘要]
- 长老C：[摘要]

准备开始第二轮Brainstorming。"
```

## 🔧 技术实现

### 长老的干活Agent

每个长老有自己的干活agent：
- 搜索相关资料
- 反驳提案
- 挑毛病
- 总结成文档

**启动方式：**
```
sessions_spawn({
  task: "作为长老A的干活agent，收到提案后：
  1. 搜索逻辑批判相关的资料
  2. 从逻辑角度质疑提案
  3. 找出逻辑漏洞
  4. 提出改进方案
  5. 生成文档
  
  通知其他长老：我的方向是[X]",
  label: "elder-a-worker",
  cleanup: "keep"
})
```

### 互相通知

长老的干活agent在开始工作时，通知其他长老：
```
message({
  action: "send",
  target: "长老B的频道",
  message: "长老A的干活agent已经开始反驳游戏化部分，方向：逻辑角度。请注意避免重复。"
})
```

## 📊 优势分析

### 对比：传统MTPM vs 三权分立

| 维度 | 传统MTPM | 三权分立 |
|------|---------|---------|
| 决策质量 | 中等（只有上议院） | 高（三院制衡） |
| 推进速度 | 快（并行执行） | 较慢（需要辩论） |
| 专业性 | 中等（一个视角） | 高（三个专业角度） |
| 盲区风险 | 高（单一视角） | 低（三个角度） |
| 决策参与度 | 低（主人被动） | 高（主人参与讨论） |
| 利用AI能力 | 中（主要是执行） | 高（思考+辩论+执行） |

### 关键优势

1. **制衡机制**
   - 不会因为某个角度太强势而忽视其他角度
   - 长老必须质疑，不能沉默

2. **专业化分工**
   - 每个长老有专业领域
   - 每个长老有干活agent
   - 效率更高

3. **互相通知**
   - 避免重复工作
   - 提高效率
   - 更好的协作

4. **主人参与度高**
   - 每个阶段都参与讨论
   - 不是命令-执行
   - 是共同决策

## 🚀 实际应用

### 番茄钟MVP开发

```
阶段1：Brainstorming（上议院）
- 讨论：MVP应该包含什么？
- 提案：纯前端游戏化番茄钟

阶段2：长老院辩论
- 长老A（逻辑）：游戏化是否分散注意力？
- 长老B（技术）：纯前端4-6周可行吗？
- 长老C（实用）：用户觉得值得下载吗？

阶段3：长老院总结
- 整合意见
- 生成修改案文档

阶段4：Brainstorming（上议院）
- 讨论：如何改进提案？
- 决策：降低游戏化，专注核心功能

阶段5：上议院决策
- 整合讨论结果
- 做出最终MVP方案

阶段6：执行（下议院）
- 执行MVP开发
```

## 📝 总结

**核心：** 三权分立 + 制衡 + 专业化

**优势：**
- ✅ 更好的决策质量（三院制衡）
- ✅ 更高的专业度（三个专业角度）
- ✅ 更好的AI能力利用（思考+辩论+执行）
- ✅ 更高的主人参与度（共同决策）

**劣势：**
- ⚠️ 推进速度较慢（需要辩论）
- ⚠️ 配置更复杂（需要3个长老agent）

**适用场景：**
- ✅ 复杂项目（需要多角度思考）
- ✅ 重要决策（需要制衡）
- ⚠️ 不适合简单紧急任务
